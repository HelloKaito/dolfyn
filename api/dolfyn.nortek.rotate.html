dolfyn.nortek.rotate.vector.beam2inst(advo, reverse=False, force=False)[source]¶

Rotate data in an ADV object from beam coordinates to instrument coordinates (or vice-versa). NOTE: this only rotates variables starting with ‘vel’ in advo.props[‘rotate_vars’].

Parameters:

advo : The adv object containing the data.

reverse : bool (default: False)

If True, this function performs the inverse rotation (principal->earth).

force : Do not check which frame the data is in prior to

performing this rotation.
dolfyn.nortek.rotate.vector.earth2principal(advo, reverse=False)[source]¶

Rotate data in an ADV object to/from principal axes. If the principal angle is not yet computed it will be computed.

All data in the advo.props[‘rotate_vars’] list will be rotated by the principal angle, and also if the data objet has an orientation matrix (orientmat) it will be rotated so that it represents the orientation of the ADV in the principal (reverse:earth) frame.

Parameters:

advo : The adv object containing the data.

reverse : bool (default: False)

If True, this function performs the inverse rotation (principal->earth).

dolfyn.nortek.rotate.vector.inst2earth(advo, reverse=False, rotate_vars=None, force=False)[source]¶

Rotate data in an ADV object to the earth from the instrument frame (or vice-versa).

Parameters:

advo : The adv object containing the data.

reverse : bool (default: False)

If True, this function performs the inverse rotation (principal->earth).

rotate_vars : iterable

The list of variables to rotate. By default this is taken from advo.props[‘rotate_vars’].

force : Do not check which frame the data is in prior to

performing this rotation.
